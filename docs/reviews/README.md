# Internal review notes

Design and code-review notes captured during development (two rounds — the
`r2_*` files are the second pass). They document *why* modules are shaped the
way they are and known trade-offs. Kept for contributors; not user
documentation. Start with [`ARCHITECTURE.md`](../ARCHITECTURE.md) and
[`FILE_FORMATS.md`](../FILE_FORMATS.md) instead if you just want to use or
extend the toolkit.
